# Roadmap

## Phase 1 – Documentation and architecture definition
Define system goals, constraints, components, data flow, and non-goals. Establish clear domain terminology and contribution standards.

## Phase 2 – Knowledge elicitation prototype
Build an initial interviewer workflow where AI asks domain experts structured questions and stores normalized outputs.

## Phase 3 – Domain module format
Define how expert knowledge is stored and loaded, including schema for concepts, relationships, rules, confidence, and provenance.

## Phase 4 – Reasoning engine
Implement a baseline inference system using graph reasoning or a probabilistic rule system with explanation output.

## Phase 5 – Domain modules
Develop and publish example domain modules such as plants, electronics, and coding to validate portability.

## Phase 6 – Evaluation
Evaluate reasoning quality, transparency, and maintainability compared to corpus-trained baseline approaches in scoped tasks.
